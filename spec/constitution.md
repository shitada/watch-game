# Kids Clock Master Constitution

## Core Principles

### I. 子供ファースト
対象年齢5〜10歳。すべてのUI・ゲームデザインは子供が直感的に
理解できることを最優先する。テキストは最小限、ひらがな・絵文字・
アニメーションで伝える。失敗にネガティブな表現は使わない。
正解・不正解ともにポジティブなフィードバックを心がける。

### II. 時計学習に集中
このゲームの目的は「時計の読み方を楽しく学ぶ」こと。
時計学習と直接関係のない機能は追加しない。
学習効果を最大化するため、レベルは段階的に難易度を上げる。

### III. デザイン
カラフルで親しみやすいデザインを基調とする。
UIは大きな角丸カードと柔らかいシャドウ。
3Dの時計はカートゥーン調で、針や文字盤が明確に見えること。

### IV. シンプル操作
iPad タッチ操作に最適化する。選択肢をタップするだけの操作。
複雑なコンボ・同時押し・長押しは一切なし。
子供が迷わず遊べるインターフェースを維持する。

### V. 技術スタック
- Three.js + TypeScript + Vite
- iPad Safari を主要ターゲットとする
- 外部ライブラリは Three.js 以外は原則追加しない
- 音声ファイル・3Dモデルファイルは使用しない（プロシージャル生成）
- YAGNI 原則を徹底する

### VI. 開発ワークフロー
- TDD で開発する
- 各ゲーム機能は独立したモジュールとして作る
- コードはシンプルで読みやすく保つ
- Vitest + jsdom でテスト
- conventional commit でコミットメッセージを書く

### VII. パフォーマンス
iPad Safari で 60fps を維持する。
Three.js オブジェクトの dispose を徹底し、メモリリークを防ぐ。

## Governance
この Constitution はプロジェクトの最上位ルールとする。
原則の変更は変更理由と影響範囲を明記した上で行う。
自動改善システムは Constitution に違反する変更を行ってはならない。
